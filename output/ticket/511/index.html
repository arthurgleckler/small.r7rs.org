Emmanuel Medernach writes:

I dislike having parameters inside the core language: however convenient there are I think promoting them is a bad idea.

Parameters can be provided portably on top of dynamic-wind provided the implementation does not provide native threads. In the presence of threads, however, such a portable implementation will produce highly unintuitive results.
